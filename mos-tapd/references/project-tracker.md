# project-tracker — 项目进度追踪

## 触发条件
- 用户说 "项目情况"、"进度如何"、"有没有风险"、"迭代完成率"、"阻塞是什么"
- 排他：只是看"个人待办" → my-todo

## 执行流程

1. **确定 workspace_id** — 按前置检查的 workspace 解析策略
2. **确定迭代**（可选）— 用户指定则传 --iteration，否则自动取当前迭代
3. **执行查询**：
   ```bash
   mos tapd skill project-tracker -w <workspace_id> [--iteration <id>]
   ```
4. **展示结果** — 健康度评分 + 进度 + 风险 + 阻塞 + 缺陷状况

## 关键命令

| 步骤 | 命令 | 说明 |
|------|------|------|
| 查询 | `mos tapd skill project-tracker -w <id>` | 默认当前迭代 |
| 指定迭代 | `mos tapd skill project-tracker -w <id> --iteration <iter_id>` | |

## 参数说明

| 参数 | 必填 | 说明 |
|------|------|------|
| -w | 是 | workspace_id |
| --iteration | 否 | 迭代 ID 或名称，默认当前迭代 |

## 健康度评分规则

| 扣分项 | 扣分 |
|--------|------|
| 每个致命/严重缺陷 | -10 |
| 每个阻塞项（>7天未更新） | -5 |
| 每个逾期需求 | -5 |

基础分 100，范围 [0, 100]。

## 健康度标识

| 评分 | 标识 | 含义 |
|------|------|------|
| >= 80 | 绿灯 | 状态良好 |
| >= 50 | 黄灯 | 需要关注 |
| < 50 | 红灯 | 存在严重风险 |

## 输出要求
- 整体健康度（评分 + 灯号）
- 迭代进度（完成率、剩余天数）
- 风险项（逾期需求列表，含负责人和逾期天数）
- 阻塞项（状态 >7 天未变更的工单）
- 缺陷状况（按严重程度分布）
