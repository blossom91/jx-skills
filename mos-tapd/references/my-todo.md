# my-todo — 个人今日待办

## 触发条件
- 用户说 "我的待办"、"今天做什么"、"工作安排"、"需要处理什么"、"梳理手头的事"
- 排他：问的是"项目整体进度" → project-tracker

## 执行流程

1. **确定 workspace_id**（可选）
   - 用户指定了项目 → 传 -w
   - 用户未指定 → 不传 -w，查所有项目的待办
2. **执行查询**：
   ```bash
   mos tapd skill my-todo [-w <workspace_id>] [--date "YYYY-MM-DD"]
   ```
3. **展示结果** — 按紧急程度排序的待办列表

## 关键命令

| 步骤 | 命令 | 说明 |
|------|------|------|
| 全部待办 | `mos tapd skill my-todo` | 查所有项目 |
| 指定项目 | `mos tapd skill my-todo -w <id>` | 只看单个项目 |
| 指定日期 | `mos tapd skill my-todo --date "2026-04-14"` | 看明天的 |

## 参数说明

| 参数 | 必填 | 说明 |
|------|------|------|
| -w | 否 | 不传则查所有项目 |
| --date | 否 | 目标日期，默认今天 |

## 多项目场景

当用户未指定项目且是**平台中心用户**时：
- 遍历 workspace-registry 中的所有项目
- 每个项目分组展示待办
- 标注项目名称

当用户未指定项目且是**其他用户**时：
- 调用 `mos tapd workspace projects` 获取项目列表
- 遍历各项目查询待办

## 排序规则
1. 逾期项最先展示（截止日期已过且未完成）
2. 优先级高的排前面（P0 > P1 > P2 > P3）
3. 截止日期近的排前面

## 输出要求
- 已逾期项（红色标注）
- 需求列表（进行中）
- 缺陷列表（待处理）
- 按紧急程度 + 截止日期排序
